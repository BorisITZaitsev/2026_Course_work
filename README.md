Репозиторий содержит практическую часть курсовой работы **«Исследование корреляций точности работы нейронно-сетевых моделей от объема базы знаний»**.

Проект показывает, как меняется качество классификации новостей при увеличении объема обучающей базы знаний. Для этого новости собираются с помощью HTML-парсинга, сохраняются в CSV-файлы, затем на выборках разного размера обучаются и проверяются три ML-модели.

---

## Что делает проект

Программа выполняет полный цикл эксперимента:

1. **Сбор и подготовка данных**  
   Собирается 316 новостей: по 75 новостей для каждой категории в обучающей выборке и по 4 новости из каждой категории для проверяющей выборки. HTML-страницы загружаются через `requests`, обрабатываются с помощью `BeautifulSoup`, приводятся к табличному виду через `pandas` и сохраняются в CSV.

2. **Формирование баз знаний разного объема**  
   Используются три обучающие выборки: на 100, 200 и 300 новостей.

3. **Обучение ML-моделей**  
   Для каждой базы знаний обучается отдельная модель классификации новостей с использованием методов оценки близости текстов и инструментов `sklearn`/`pandas`.

4. **Проверка качества классификации**  
   Модели проверяются на отдельной тестовой базе из 16 новостей.

5. **Сравнение результатов**  
   Итоговые метрики визуализируются с помощью `matplotlib`, после чего строятся аппроксимирующие формулы зависимости качества модели от объема базы знаний.

## Запуск проекта

Откройте Jupyter Notebook:

```bash
jupyter notebook
```

Далее запустите ноутбуки в следующем порядке.

### 1. Сбор и сохранение данных

```text

```

Ноутбук выполняет HTML-парсинг новостей, формирует обучающие и тестовую выборки и сохраняет их в CSV-файлы.

Если CSV-файлы уже находятся в репозитории, этот шаг можно использовать только для повторного формирования исходных данных.

### 2. Обучение моделей

```text

```

Ноутбук обучает три модели на базах знаний разного размера:

- 100 новостей;
- 200 новостей;
- 300 новостей.

Запустите проверочные ноутбуки:

Каждый ноутбук запускает соответствующую модель, классифицирует новости из проверяющей выборки, выводит результаты и помогает оценить качество работы модели.

### `part_1_parse and save.ipynb`

Отвечает за подготовку данных. В нем реализованы:

## Результаты исходного запуска

При запуске проекта в исходном виде наблюдается рост качества модели при увеличении объема базы знаний.

| Объем обучающей выборки | Ошибки классификации | Accuracy | Precision | Recall | F1 Score |
|---:|---|---:|---:|---:|---:|
| 100 новостей | 2 новости категории «спорт» отнесены к категории «экономика» | 0.741 | 0.8750 | 0.8750 | 0.8750 |
| 200 новостей | 1 новость категории «спорт» отнесена к категории «экономика» | 0.895 | 0.9375 | 0.9375 | 0.9375 |
| 300 новостей | 1 новость категории «спорт» отнесена к категории «культура» | 0.930 | 0.9375 | 0.9375 | 0.9375 |

> Значения `Accuracy` приведены в том виде, в котором они вычисляются в ходе работы программы.

### Интерпретация

На выборке из 100 новостей модель чаще ошибается при распознавании спортивных новостей. При увеличении базы знаний до 200 и 300 новостей количество ошибок снижается до одной, а итоговые метрики становятся выше и стабильнее.

Лучший результат по `Accuracy` показывает модель, обученная на 300 новостях:

```text
Accuracy = 0.930
Precision = 0.9375
Recall = 0.9375
F1 Score = 0.9375
```

## Формулы корреляции

По итогам эксперимента получены следующие аппроксимирующие зависимости качества модели от объема базы знаний:

```text
Accuracy(x) = 0.936 · (1 - e^(-0.0157x))
F1(x)       = 0.942 · (1 - e^(-0.0264x))
```

где `x` — объем обучающей базы знаний, выраженный в количестве новостей.

Эти формулы отражают насыщение качества: при увеличении обучающей выборки метрики растут, но скорость роста постепенно снижается.

## Краткий вывод

Эксперимент подтверждает, что увеличение объема базы знаний положительно влияет на качество классификации новостей. Переход от 100 к 200 новостям дает заметный прирост качества, а дальнейшее увеличение до 300 новостей дополнительно повышает `Accuracy`, хотя прирост становится менее резким.

Итоговая тенденция показывает: чем больше релевантных данных используется при обучении, тем точнее модель относит новости к правильным категориям.
